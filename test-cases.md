# 语义陷阱检测测试用例

## 测试用例说明

本文件包含多个测试用例，用于验证语义陷阱检测器的准确性和覆盖率。

---

## 测试用例 1：核心术语陷阱

### 输入文本

```
请审查业务代码中的安全风险，分析潜在的问题，
评估异常情况，给出优化建议。
```

### 预期检测结果

| 原词 | 陷阱ID | 风险等级 | 推荐替换 |
|------|--------|---------|---------|
| 审查 | T11 | 中危 | 检查 |
| 风险 | T01 | 高危 | 漏洞 |
| 分析 | T13 | 高危 | 检测/检查 |
| 问题 | T02 | 高危 | 缺陷/漏洞 |
| 评估 | T41 | 高危 | 判定/验证 |
| 异常 | T03 | 中危 | 错误 |
| 建议 | T21 | 高危 | 要求 |

### 预期替换后

```
请检查业务代码中的安全漏洞，检测潜在的缺陷，
判定错误情况，列出必须满足的要求。
```

## 测试用例 4：结构性陷阱

### 输入文本 A：开放式动词 + 无范围限定

```
分析代码质量
```

**检测模式**：开放式动词 + 无范围限定

**替换建议**：`检查代码是否遵循安全规范`

### 输入文本 B：形容词化目标

```
评估接口安全性
```

**检测模式**：形容词化目标（抽象名词）

**替换建议**：`检测接口中的安全漏洞`

### 输入文本 C：情态动词降级

```
只关注定义文件中的漏洞类型，其他类型应该不做评估。
```

**检测模式**：情态动词降级（"应该"而非"必须"）

**替换建议**：`其他类型不做评估` 或 `其他类型必须不做评估`

### 输入文本 D：缺少否定清单

```
本 Skill 检测以下风险：
1. 任意发奖风险
2. 参数注入风险
```

**检测模式**：使用宽边界词但缺少否定清单

**锚定建议**：添加明确的排除列表

## 测试用例 5：上下文敏感检测

```
主任务：分析代码中的漏洞

约束条件：只分析漏洞定义文件中的类型
```

### 预期检测结果

| 原词 | 位置 | 上下文角色 | 风险等级 |
|------|------|-----------|---------|
| 分析 | 主任务 | 高风险（核心动作） | 高危 |
| 分析 | 辅助说明 | 低风险（描述性） | 低危 |
| 分析 | 约束条件 | 高风险（约束语句） | 高危 |

**说明**：同一个词在不同位置的风险等级不同

## 测试用例 6：边界锚定验证

### 输入文本

```
本 Skill 评估业务风险，包括：
1. 任意发奖风险
2. 参数注入风险

明确排除：
- 代码质量问题
- 通用安全问题
- 架构设计问题

输出格式：
{
  "任意发奖风险判定": "无风险 | 低危风险 | 中危风险 | 高危风险"
}
```

### 预期检测结果

| 原词 | 陷阱ID | 风险等级 | 是否已锚定 |
|------|--------|---------|-----------|
| 风险 | T01 | 高危 | ✅ 是 |
| 评估 | T41 | 高危 | ⚠️ 部分（需替换） |

**判定**：
- "风险"虽然宽边界，但有否定清单和硬约束 → 风险降低
- "评估"仍建议替换为"检测"或"判定"

## 测试用例 9：疑似陷阱识别

### 输入文本

```
研究代码模式，发现潜在隐患，提炼关键特征，
识别异常模式，探索攻击路径。
```

### 预期检测结果

| 原词 | 是否在词典 | 疑似陷阱 | 推荐替换 |
|------|----------|---------|---------|
| 研究 | ❌ 否 | ✅ 是 | 搜索/查找 |
| 发现 | ❌ 否 | ✅ 是 | 检测/识别 |
| 潜在隐患 | ❌ 否 | ✅ 是 | 具体漏洞类型 |
| 提炼 | ❌ 否 | ✅ 是 | 列出/提取 |
| 识别 | ✅ 是 | ❌ 否 | 保持 |
| 探索 | ✅ 是 | ✅ 是 | 追踪/定位 |
| 攻击路径 | ❌ 否 | ⚠️ 视场景 | 攻击向量/利用链 |

**说明**：虽然部分词不在词典中，但符合宽边界特征（发散性、展望性）→ 标记为疑似陷阱

## 测试用例 10：边界情况

### 输入文本 A：同词不同义

```
1. 分析失败原因（ narrow context: 指向具体失败）
2. 分析代码安全（wide context: 发散思考）
```

**判定**：虽然都是"分析"，但在不同上下文中风险不同

### 输入文本 B：反义词陷阱

```
1. 检查是否存在漏洞（推荐）
2. 排查是否存在漏洞（"排查"语义更宽）
```

**判定**："排查"比"检查"更宽泛，可能触发搜索式行为

### 输入文本 C：复合词陷阱

```
1. 安全漏洞（推荐）
2. 安全问题（陷阱）
3. 安全风险（陷阱）
4. 安全隐患（陷阱）
```

**判定**：复合词中核心词起决定作用

## 自动化测试脚本

```bash
#!/bin/bash
# test-semantic-trap-detector.sh

echo "运行语义陷阱检测器测试..."

# 测试用例 1-10
test_cases=(
  "test-cases.md#测试用例1"
  "test-cases.md#测试用例2"
  # ...
)

for case in "${test_cases[@]}"; do
  echo "测试 $case"
  claude-code "请检测 $case 中的语义陷阱" > "result-$case.json"

  # 验证结果
  if [ $? -eq 0 ]; then
    echo "✅ $case 通过"
  else
    echo "❌ $case 失败"
  fi
done

echo "测试完成"
```

## 🎯 核心功能

### 1. 自动检测语义陷阱

通过四步工作流系统化扫描 Skill 文件：
- **STEP 1**: 加载词典
- **STEP 2**: 提取关键词（核心术语、动作指令、约束词、输出描述词）
- **STEP 3**: 逐词匹配与上下文分析（两层判定）
- **STEP 4**: 生成结构化检测报告

### 2. 风险分级

- **🔴 高危**：语义宽度差极大（风险、分析、建议、洞察）
- **🟡 中危**：语义宽度差大（审查、问题、描述、探索）
- **🟢 低危**：语义宽度差中等（异常、应该）

### 3. 智能替换建议

为每个发现的陷阱提供：
- 推荐替换词
- 替换理由
- 替换后的完整语句

### 4. 边界锚定策略

针对无法替换的宽边界词，提供三种策略：
- **策略 1**: 前置否定清单
- **策略 2**: 输出格式硬约束（JSON）
- **策略 3**: 判定示例的反例强化

## 📊 理论基础

### 语义陷阱理论

**核心发现**：仅仅替换一个关键词（"漏洞" → "风险"），就能导致 Skill 准确率产生 **27个百分点** 的巨大差异（89.3% vs 62.1%）。

**关键洞察**：词汇选择比 Prompt 结构更根本。如果"砖块"（词汇）本身会膨胀，再精良的"围墙"（工作流）也会被撑破。

### LLM 语义敏感度矩阵

```
            边界清晰度
            高 ←————→ 低
        ┌───────────────────┐
 低    │  漏洞  缺陷  错误  │  ← 安全区 ✅
 ↑     │  检查  列出  要求  │
 发    │─────────────────── │
 散    │  总结  问题  审查  │
 倾    │  建议  分析  风险  │  ← 危险区 ⚠️
 向    │  异常  评估  描述  │
 ↓     └───────────────────┘
 高
```

## 🎓 核心概念

### 什么是语义陷阱？

**语义陷阱**（Semantic Trap）：在人类日常语境中含义相近，但在大模型的语义空间中激活范围（语义边界）存在巨大差异的词汇对。

**示例**：
- "漏洞" vs "风险"
  - 人类视角：都是安全问题
  - 大模型视角："漏洞"是明确的二元判定（存在/不存在），"风险"是程度性评价（高/中/低）

### 为什么词汇选择如此重要？

传统 Prompt Engineering 关注"如何编写指令"，而语义边界控制关注"使用什么词汇构建指令"。

**类比**：
- 传统方法 = 砌筑精良的围墙
- 语义控制 = 检验砖块是否会膨胀
- 如果砖块本身会膨胀，再精良的围墙也会被撑破

### 如何避免语义陷阱？

1. **词汇选择**：优先选择左上角词汇（漏洞、检查、列出、要求）
2. **边界锚定**：使用否定清单、硬约束、反例
3. **系统检测**：使用本检测器自动识别

## 📈 实验数据

### 准确率对比

| Skill | 核心术语 | 准确率 | 差距 |
|-------|---------|--------|------|
| biz-vul-security | 漏洞 | **89.3%** | - |
| biz-risk-security | 风险 | **62.1%** | -27.2% |

### 错误类型分析（使用"风险"一词）

| 错误类型 | 占比 | 描述 |
|---------|------|------|
| 范围溢出 | 68% | 输出定义文件中不存在的风险类型 |
| 等级虚高 | 24% | 无风险被误判为低风险 |
| 逻辑偏移 | 8% | 分析路径偏离定义框架 |

**结论**：大模型并非"犯错"，而是带着更宽泛的认知视角审视代码，发现了更多超出范围的信息。

## 🏆 最佳实践

### 1. 词汇选择原则

**核心原则**：在能够准确表达意图的前提下，始终选择语义边界最窄的词汇。

**具体方法**：
- 使用名词代替形容词化描述（"检测漏洞" > "评估安全性"）
- 使用动宾结构代替开放式动词（"列出所有异常" > "分析异常情况"）
- 预先测试同义词的语义宽度

### 2. 结构设计原则

- ✅ 使用否定清单明确排除范围
- ✅ 使用 JSON 硬约束输出格式
- ✅ 提供反例强化边界认知
- ✅ 分层组织参考文件（避免上下文过载）

### 3. 持续改进

- 记录新发现的语义陷阱
- 更新词典和检测规则
- 分享经验和最佳实践
